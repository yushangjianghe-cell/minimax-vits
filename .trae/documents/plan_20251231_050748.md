# 修复Koishi客户端无法产生可编辑UI的问题

## 问题分析
1. **缺少控制台UI组件注册**：插件没有注册任何Koishi控制台UI组件
2. **配置面板未注册**：虽然定义了Config Schema，但没有与控制台UI关联
3. **缺少ChatLuna UI集成**：ChatLuna可能需要额外配置才能显示可编辑选项
4. **Console Service功能不完整**：当前的MinimaxVitsService只是一个基础服务，没有UI相关功能

## 修复方案

### 1. 修复Console Service注册
- 确保Console Service正确注册到Koishi控制台
- 添加必要的UI组件支持

### 2. 添加配置面板注册
- 使用Koishi的控制台API注册配置面板
- 确保配置项在控制台中可编辑

### 3. 完善ChatLuna集成
- 检查ChatLuna插件的UI要求
- 确保ChatLuna工具配置正确，包括authorization参数

### 4. 添加必要的UI组件
- 为插件添加基础的控制台UI组件
- 确保UI组件能够正确显示和编辑配置

## 具体修改步骤

1. **检查Koishi控制台API使用**：确保正确使用Koishi控制台的addPanel、addComponent等API
2. **修复Config Schema与UI的关联**：确保配置项能够在控制台中正确显示
3. **完善MinimaxVitsService**：添加UI相关的服务方法
4. **检查ChatLuna集成配置**：确保ChatLuna工具的authorization配置正确
5. **添加控制台UI组件**：添加必要的UI组件文件

## 预期效果
- 插件在Koishi控制台中显示可编辑的配置面板
- 用户可以在控制台中修改插件配置
- ChatLuna集成正常，显示可编辑的选项
- 控制台服务正常工作，支持UI交互
